# ドキュメント更新手順

ドキュメントは [MkDocs Material](https://squidfunk.github.io/mkdocs-material/) で生成され、
`main` ブランチへの push 時に GitHub Actions が自動で [GitHub Pages](https://hiraoka-group.github.io/nasap-fit/) へデプロイする。

---

## ローカルでのプレビュー

```bash
# 依存ライブラリのインストール（初回のみ）
pip install -r requirements-docs.txt

# 開発サーバー起動（ファイルを保存するたびに自動リロード）
mkdocs serve
```

ブラウザで http://127.0.0.1:8000 を開くと確認できる。

---

## 更新したい内容ごとの対応ファイル

| 更新したい内容 | 対応ファイル |
|---|---|
| トップページ | `docs/index.md` |
| Getting Started（設定・ワークフロー解説） | `docs/getting-started.md` |
| API リファレンス（自動生成） | `nasap_fit/api.py` の docstring |
| API リファレンスのページ構成 | `docs/api-reference.md` |
| サイト全体の設定（テーマ・ナビ等） | `mkdocs.yml` |
| カスタム CSS | `docs/stylesheets/extra.css` |

### API リファレンスについて

`docs/api-reference.md` に書かれた `:::` ディレクティブ（mkdocstrings）が
`nasap_fit/api.py` の docstring を読み込んで HTML を生成する。
**API の説明を変えたい場合は `api.py` の docstring を編集すればよい。**

---

## デプロイ

`main` ブランチに push（または PR をマージ）すると GitHub Actions が自動でビルド・デプロイする。
手動でデプロイしたい場合は GitHub リポジトリの Actions タブ →「Deploy docs to GitHub Pages」→「Run workflow」を押す。

> **注意:** `site/` ディレクトリはビルド生成物のため `.gitignore` 済み。コミットしないこと。
